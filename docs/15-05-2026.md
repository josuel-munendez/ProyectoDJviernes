Los tipos más comunes de inyección en el código son: 

- Inyección SQL (SQLi): El atacante inserta comandos SQL en las consultas de la base de datos a través de los campos de entrada de la aplicación. Permite leer, modificar o borrar información confidencial. 

- Secuencias de comandos entre sitios (XSS): Se inyectan scripts maliciosos (generalmente JavaScript) en páginas web vistas por otros usuarios. 

- Inyección de Comandos (OS Command Injection): El atacante ejecuta comandos arbitrarios del sistema operativo directamente en el servidor que aloja la aplicación. 

- Inyección LDAP: Dirigida a aplicaciones que usan servicios de directorio (LDAP) para manipular los resultados de las consultas de autenticación o búsqueda. 

- Inyección HTML: Inserta código HTML malicioso para alterar la apariencia de una web o realizar acciones malintencionadas. 

- Inyección de Código/Scripting (PHP, ASP, etc.): Ocurre cuando la aplicación pasa datos no validados a una función que ejecuta código, como `eval()` en PHP, permitiendo al atacante ejecutar código arbitrario. 

- Inyección CRLF: Utiliza caracteres de retorno de carro y salto de línea ( `\r\n` ) para manipular cabeceras HTTP o inyectar respuestas falsas. 

- Inyección XPath: Similar a SQLi, pero diseñada para consultar y manipular estructuras de documentos XML. [1, 2, 3, 4, 5] 

## Medidas de prevención clave: 

- Validación y saneamiento de entradas: No confiar en ningún dato proveniente del usuario. 

- Consultas parametrizadas: Usar sentencias preparadas para separar el código de los datos en consultas SQL. 

- Escapado de caracteres: Escapar caracteres especiales antes de utilizarlos en comandos o consultas. [1, 2, 3, 4, 5]

Una solicitud HTTP es un mensaje estructurado que un cliente envía a un servidor web para solicitar un recurso o activar una acción específica. Es la base de la comunicación en la arquitectura de Internet. [1, 2, 3, 4] 

## Componentes de una Solicitud HTTP 

Cada mensaje de solicitud enviado al servidor incluye partes bien definidas: [1] 

- Línea de solicitud: Contiene el método HTTP, la URL del recurso y la versión del protocolo. 

- Encabezados (Headers): Metadatos con información sobre el cliente, el formato de datos aceptado y cookies. 

- Cuerpo (Body): Bloque de datos opcional que se envía al servidor cuando se crean o modifican recursos. [1, 2, 3, 4, 5] 

## Métodos HTTP más Comunes 

El protocolo emplea verbos estandarizados para definir la naturaleza de la operación: [1] 

1. `GET` : Solicita datos de un recurso específico sin alterar el estado del sistema. 

2. `POST` : Envía datos al servidor para crear un nuevo registro o recurso. 

3. `PUT` : Reemplaza por completo el estado del recurso existente con los nuevos datos provistos. 

4. `PATCH` : Aplica modificaciones parciales y específicas a un recurso ya establecido. 

5. `DELETE` : Solicita la eliminación definitiva del recurso indicado en el servidor. [1, 2,
